## Core Design Patterns

### 1. Singleton Factory Pattern

The template uses a singleton factory pattern for server management:

```python
# Base factory class
class BaseServerFactory:
    _instance = None
    
    @classmethod
    def create(cls, **kwargs):
        """Create singleton instance"""
        if cls._instance is not None:
            raise AssertionError("not allowed to create more than one instance")
        cls._instance = cls._create(**kwargs)
        return cls._instance
    
    @classmethod
    def get(cls):
        """Get existing instance"""
        if cls._instance is None:
            raise AssertionError("Please create a FastAPI instance first")
        return cls._instance
    
    @classmethod
    def reset(cls):
        """Reset for testing"""
        cls._instance = None
```

### 2. Modular Architecture

The project is organized into distinct modules:

- **MCP Module** (`src/mcp/`): Handles MCP server creation and transport management
- **Web Server Module** (`src/web_server/`): Manages FastAPI web server and API endpoints
- **Integration Module** (`src/integrate/`): Combines MCP and web server into unified application
- **Models Module** (`src/models/`): Contains domain models and DTOs

### 3. Transport Abstraction

The MCP server supports multiple transport modes:

```python
class MCPTransportType(str, Enum):
    """MCP transport types"""
    SSE = "sse"                      # Server-Sent Events
    HTTP_STREAMING = "streamable-http"  # HTTP streaming
    STDIO = "stdio"                  # Standard I/O
```

## Component Interactions

### Server Creation Flow

```
Entry Point (entry.py)
    ↓
Parse Arguments & Configuration
    ↓
Choose Server Mode (Standalone or Integrated)
    ↓
├─ Standalone Mode
│   ├─ Create MCP Server (MCPServerFactory)
│   └─ Run with selected transport
│
└─ Integrated Mode
    ├─ Create MCP Server (MCPServerFactory)
    ├─ Create Web Server (WebServerFactory)
    └─ Mount MCP on Web Server (IntegratedServerFactory)
```

### Lifecycle Management

Each factory manages its component's lifecycle:

```python
# MCP Server Lifecycle
mcp_factory.create()  # Initialize MCP server
mcp_server = mcp_factory.get()  # Access server
mcp_factory.reset()  # Cleanup for testing

# Web Server Lifecycle
web_factory.create()  # Initialize FastAPI app
web_app = web_factory.get()  # Access app
web_factory.reset()  # Cleanup for testing

# Integrated Server Lifecycle
with integrated_server_lifecycle():
    app = integrated_factory.create()
    # Use app
    # Cleanup happens automatically
```

## Configuration Management

Configuration is handled through multiple layers:

1. **Environment Variables**: Loaded from `.env` file
2. **CLI Arguments**: Parsed from command-line input
3. **Settings Object**: Centralized configuration via `ServerConfig`

```python
# Configuration flow
load_env_vars()
    ↓
parse_cli_args()
    ↓
create_server_config()
    ↓
initialize_servers()
```

## MCP Server Implementation

### FastMCP Integration

The template uses FastMCP for MCP server implementation:

```python
from fastmcp import FastMCP

mcp = FastMCP(name="my-server")

@mcp.tool()
def my_tool(input: MyToolInput) -> MyToolOutput:
    """Tool implementation"""
    return MyToolOutput(...)
```

### Transport Modes

#### SSE (Server-Sent Events)
- Unidirectional HTTP communication
- Good for browser-based clients
- Mount path: `/mcp/sse`

#### HTTP Streaming
- Bidirectional HTTP streaming
- Robust HTTP-based communication
- Mount path: `/mcp`

#### Stdio
- Standard input/output communication
- Used for direct process communication
- No HTTP overhead

## Web Server Implementation

### FastAPI Integration

The web server is built on FastAPI:

```python
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware

app = FastAPI(title="MCP Server")

# CORS configuration
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

# Health check endpoint
@app.get("/health")
async def health_check():
    return {"status": "healthy"}
```

### API Structure

```
/health                 # Health check endpoint
/mcp                    # MCP transport mount point
  ├── /sse             # SSE transport
  └── /http            # HTTP streaming transport
```

## Extension Points

### Adding Custom Tools

```python
from src.mcp import mcp_factory
from src.mcp.models.input.my_tool_input import MyToolInput
from src.mcp.models.output.my_tool_output import MyToolOutput

@mcp_factory.get().tool()
def my_custom_tool(input: MyToolInput) -> MyToolOutput:
    """Custom tool implementation"""
    return MyToolOutput(...)
```

### Adding Custom Endpoints

```python
from fastapi import APIRouter
from src.web_server import web_factory
from src.web_server.models.request.my_request import MyRequest
from src.web_server.models.response.my_response import MyResponse

router = APIRouter(prefix="/api", tags=["custom"])

@router.post("/custom", response_model=MyResponse)
async def custom_endpoint(request: MyRequest) -> MyResponse:
    """Custom endpoint implementation"""
    return MyResponse(...)

# Register router
app = web_factory.get()
app.include_router(router)
```

### Extending Configuration

```python
from src.models import ServerConfig

class CustomServerConfig(ServerConfig):
    """Extended configuration"""
    custom_field: str = "default_value"
    
    def get_custom_setting(self) -> str:
        return self.custom_field
```

## Error Handling

### Validation Errors

Pydantic models provide automatic validation:

```python
from pydantic import ValidationError

try:
    input_data = MyToolInput(invalid_field="value")
except ValidationError as e:
    print(e.json())  # Detailed error information
```

### Server Errors

Clear error messages for common issues:

```python
# Singleton violation
AssertionError: "not allowed to create more than one instance"

# Invalid transport
ValueError: "Invalid transport type for integrated server"

# Server not created
AssertionError: "Please create a FastAPI instance first"
```

## Performance Considerations

### Dependency Resolution

- **uv**: 10-100x faster than pip for dependency resolution
- **Lock file**: Deterministic builds with `uv.lock`

### Async Support

- FastAPI uses async/await for high concurrency
- Pytest configured with `pytest-asyncio` for async tests

### Caching

- Configuration is cached after first load
- Factory instances are singletons (created once)

## Best Practices

1. **Use type hints** for all functions and variables
2. **Validate inputs** with Pydantic models
3. **Document code** with docstrings and comments
4. **Test thoroughly** with unit and integration tests
5. **Follow naming conventions** (snake_case for functions/variables)
6. **Keep modules focused** on single responsibility
7. **Use dependency injection** for testability
8. **Handle errors gracefully** with meaningful messages
