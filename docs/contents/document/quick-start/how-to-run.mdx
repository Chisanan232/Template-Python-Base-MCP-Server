# Setting Up Your MCP Server

This guide explains how to customize and configure your new MCP server project after creating it from the template.

## Step 1: Update Project Metadata

After creating your project from the template, customize the `pyproject.toml` file with your MCP server details:

```toml title="pyproject.toml"
[project]
name = "your-mcp-server"  # ← Update this
version = "0.1.0"  # ← Set initial version
description = "Your MCP server description"  # ← Update this
authors = [{ name = "Your Name", email = "your.email@example.com" }]  # ← Update this
requires-python = ">=3.12,<4.0"
readme = "README.md"
license = { file = "LICENSE" }
keywords = ["mcp", "server", "ai", "model-context-protocol"]  # ← Add relevant keywords
```

## Step 2: Set Up Development Environment

### Install Dependencies and Activate Environment

```bash
# Install all dependencies (including dev tools)
uv sync

# Activate the virtual environment (optional, uv run handles this automatically)
source .venv/bin/activate  # macOS/Linux
# or
.venv\Scripts\activate     # Windows
```

### Set Up Pre-commit Hooks

```bash
# Install pre-commit hooks for code quality
uv run pre-commit install

# Test pre-commit setup
uv run pre-commit run --all-files
```

## Step 3: Configure Your MCP Server

### Set Up Environment Configuration

Copy the example environment file and configure your MCP server:

```bash
# Copy the example environment file
cp .env.example .env
```

Edit the `.env` file with your MCP server configuration:

```bash title=".env"
# =============================================================================
# API Configuration
# =============================================================================

# API authentication token (required for most operations)
API_TOKEN=your_mcp_server_token_here

# =============================================================================
# Server Configuration
# =============================================================================

# Server host address (default: 127.0.0.1)
# Use 0.0.0.0 to bind to all interfaces (accessible from other machines)
# Use 127.0.0.1 to bind only to localhost (only accessible from this machine)
HOST=127.0.0.1

# Server port number (default: 8000)
PORT=8000

# Logging level (default: info)
# Available levels: debug, info, warning, error, critical
LOG_LEVEL=info

# MCP transport protocol (default: sse)
# Available transports: sse, http-streaming
TRANSPORT=sse

# =============================================================================
# CORS Configuration
# =============================================================================

# CORS allowed origins (default: ["*"])
# For development, you might want: ["http://localhost:3000", "http://localhost:8080"]
# For production, use specific domains: ["https://yourdomain.com"]
CORS_ALLOW_ORIGINS=["*"]
```

### Understanding MCP Server Modes

Your MCP server supports two main modes:

<Tabs>
<TabItem value="standalone" label="Standalone Mode">

**Standalone Mode**: Runs only the MCP server

```bash
# Run standalone MCP server with SSE transport
python -m src.entry --transport sse --port 8000

# Run with HTTP streaming transport
python -m src.entry --transport http-streaming --port 8000
```

**Use Case**: When you only need MCP protocol functionality

</TabItem>
<TabItem value="integrated" label="Integrated Mode">

**Integrated Mode**: Runs both MCP server and webhook server

```bash
# Run integrated server (MCP + webhook)
python -m src.entry --integrated --transport sse --port 8000
```

**Use Case**: When you need both MCP protocol and webhook endpoints

</TabItem>
</Tabs>

## Step 4: MCP Server Development Workflow

### Running Tests

```bash
# Run all tests
uv run pytest

# Run tests with coverage
uv run pytest --cov=src --cov-report=html

# Run specific test file
uv run pytest test/unit_test/test_models.py
```

### Code Quality Checks

```bash
# Format code with black
uv run black src/ test/

# Lint code with ruff
uv run ruff check src/ test/

# Type checking with mypy
uv run mypy src/

# Run all pre-commit hooks
uv run pre-commit run --all-files
```

### Testing Your MCP Server

<Tabs>
<TabItem value="health-check" label="Health Check">

```bash
# Start your MCP server
python -m src.entry --transport sse --port 8000

# Test health endpoint in another terminal

```

Expected response:
```json
{"status": "healthy", "timestamp": "2024-01-01T12:00:00Z"}
```

</TabItem>

</TabItem>

## Step 5: Configure CI/CD

The template includes GitHub Actions workflows in `.github/workflows/`:

### Update GitHub Secrets (if needed)

For advanced workflows, you may need to configure repository secrets:

1. Go to your GitHub repository → Settings → Secrets and variables → Actions
2. Add any required secrets (e.g., PyPI tokens, deployment keys)

### Customize Workflows

Edit `.github/workflows/ci.yaml` to match your project needs:

```yaml title=".github/workflows/ci.yaml"
name: CI
on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        python-version: ["3.12", "3.13"]  # Customize Python versions
    
    steps:
      - uses: actions/checkout@v4
      - name: Set up uv
        uses: ./.github/actions/setup-python-uv
        with:
          python-version: ${{ matrix.python-version }}
      
      # Your custom test steps here
```

## Step 6: Documentation Setup

### Docusaurus Documentation (Optional)

```bash
# Navigate to documentation directory
cd docs_with_docusarus

# Install documentation dependencies
npm install

# Start development server
npm start

# Build static documentation
npm run build
```

### MkDocs Documentation (Alternative)

```bash
# Navigate to MkDocs directory
cd docs_with_mkdocs

# Install MkDocs dependencies
uv run pip install -r requirements.txt

# Start development server
uv run mkdocs serve

# Build documentation
uv run mkdocs build
```

## Step 7: Final MCP Server Setup

### Initialize Git Repository

```bash
# If you used Method 2 (manual setup) from installation
git add .
git commit -m "Initial commit from MCP Server template"

# Connect to remote repository
git remote add origin https://github.com/YOUR-USERNAME/YOUR-MCP-SERVER.git
git push -u origin main
```

### Clean Up Template References

1. Remove or update template-specific files you don't need
2. Update documentation to reflect your MCP server
3. Customize GitHub issue templates in `.github/ISSUE_TEMPLATE/`

## Running Your MCP Server

After customization, test that everything works:

```bash
# Run your MCP server with default settings
python -m src.entry

# Run with specific transport and port
python -m src.entry --transport sse --port 8000

# Run in integrated mode with debug logging
python -m src.entry --integrated --log-level debug --reload
```

### Verify MCP Server Functionality

```bash
# Test all endpoints
curl http://localhost:8000/health  # Health check
curl http://localhost:8000/sse      # SSE transport
curl http://localhost:8000/mcp      # HTTP streaming

# Check server logs for proper MCP protocol handling
```
