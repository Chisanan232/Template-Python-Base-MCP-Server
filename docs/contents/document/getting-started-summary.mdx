# Getting Started with MCP Server Template

Welcome to the MCP Server Template! This document provides a quick overview of what this template offers and how to get started.

## What is This Template?

The **MCP Server Template** is a production-ready Python project template for building Model Context Protocol (MCP) servers. It combines:

- **MCP Server**: FastMCP-based server for tool implementation
- **Web Server**: FastAPI-based HTTP API with health checks
- **Integration Layer**: Unified server combining both MCP and web functionality
- **Modern Tooling**: uv package management, pytest testing, type checking, and more

## Key Features

✅ **Multiple Transport Modes**: SSE, HTTP Streaming, and Stdio support  
✅ **Modular Architecture**: Clean separation of concerns with MCP, Web, and Integration layers  
✅ **Type Safety**: Full type hints and Pydantic validation  
✅ **Comprehensive Testing**: Unit and integration tests with pytest  
✅ **Production Ready**: Docker support, health checks, monitoring  
✅ **Well Documented**: Extensive documentation and examples  

## Quick Start (5 Minutes)

### 1. Install Dependencies

```bash
# Install uv (if not already installed)
curl -LsSf https://astral.sh/uv/install.sh | sh

# Clone and setup project
git clone <repository-url>
cd mcp-server-template
uv sync
```

### 2. Run the Server

```bash
# Run integrated server (MCP + Web API)
uv run python -m src.entry --integrated

# Server will be available at:
# - API Docs: http://localhost:8000/docs
# - Health Check: http://localhost:8000/health
# - MCP Endpoint: http://localhost:8000/mcp
```

### 3. Test the Server

```bash
# In another terminal, test health endpoint
curl http://localhost:8000/health

# Expected response:
# {"status": "healthy", "timestamp": "...", "version": "0.1.0"}
```

## Common Tasks

### Adding a Custom Tool

1. **Create input/output models** in `src/mcp/models/input/` and `src/mcp/models/output/`
2. **Implement tool** in `src/mcp/tools/`
3. **Register tool** by importing in `src/mcp/__init__.py`
4. **Test tool** with pytest

See [Examples & Tutorials](./examples.mdx) for detailed examples.

### Adding a Custom API Endpoint

1. **Create request/response models** in `src/web_server/models/request/` and `src/web_server/models/response/`
2. **Create router** in `src/web_server/routers/`
3. **Register router** with web server
4. **Test endpoint** with pytest

See [Usage Guide](./usage-guide.mdx) for detailed instructions.

### Running Tests

```bash
# Run all tests
uv run pytest

# Run with coverage
uv run pytest --cov=src --cov-report=html

# Run specific test
uv run pytest test/unit_test/mcp/test_calculator.py -v
```

### Deploying to Production

```bash
# Build Docker image
docker build -t my-mcp-server .

# Run container
docker run -p 8000:8000 my-mcp-server

# Or use Docker Compose
docker-compose up -d
```

See [Deployment Guide](./deployment.mdx) for detailed deployment instructions.

## Running Different Server Modes

### Standalone MCP Server

```bash
# Run with SSE transport (default)
uv run python -m src.entry --mcp-transport sse

# Run with HTTP streaming
uv run python -m src.entry --mcp-transport streamable-http

# Run with stdio
uv run python -m src.entry --mcp-transport stdio
```

### Web Server Only

```bash
# Run web server with health check and API endpoints
uv run python -m src.entry --web-only
```

### Integrated Server (Recommended)

```bash
# Run both MCP and web server together
uv run python -m src.entry --integrated

# With custom settings
uv run python -m src.entry --integrated --port 8001 --mcp-mount-path /api/mcp
```

## Key Concepts

### MCP (Model Context Protocol)

MCP is a protocol for connecting AI models to tools and data sources. This template helps you build MCP servers that expose tools to AI models.

### Tools

Tools are functions that MCP clients can call. Each tool has:
- **Input Model**: Defines parameters and validation
- **Output Model**: Defines return value structure
- **Implementation**: The actual function logic

### Transports

MCP supports different communication protocols:
- **SSE**: Server-Sent Events (unidirectional)
- **HTTP Streaming**: Bidirectional HTTP streaming
- **Stdio**: Standard input/output

### Web Server

The integrated web server provides:
- **Health Checks**: Monitor server status
- **API Endpoints**: Custom HTTP endpoints
- **CORS Support**: Cross-origin requests
- **API Documentation**: Automatic Swagger/ReDoc docs
