## Key Concepts Explained

### MCP (Model Context Protocol)

MCP is a protocol for connecting AI models to tools and data sources. This template helps you build MCP servers that expose tools to AI models.

**Learn more:** See [Architecture Overview - MCP Server Implementation](./architecture.mdx#mcp-server-implementation)

### Singleton Factory Pattern

A design pattern used to ensure only one instance of a server component exists. This template uses it for MCP, Web, and Integrated servers.

**Learn more:** See [Architecture Overview - Singleton Factory Pattern](./architecture.mdx#1-singleton-factory-pattern)

### Transport Modes

MCP supports different communication protocols:
- **SSE (Server-Sent Events)**: Unidirectional HTTP communication
- **HTTP Streaming**: Bidirectional HTTP streaming
- **Stdio**: Standard input/output communication

**Learn more:** See [Architecture Overview - Transport Abstraction](./architecture.mdx#3-transport-abstraction)

### Pydantic Models

Used for data validation and serialization. All inputs and outputs use Pydantic models for type safety.

**Learn more:** See [Best Practices - Pydantic Models](./best-practices.mdx#pydantic-models)
